## Global Design System Rules (Applied to ALL phases)

> [!IMPORTANT]
> These rules are non-negotiable and must be enforced in every component.

- **Theme:** "Industrial Dark" — never use pure `#000000`
- **Backgrounds:** `bg-slate-950` for body, `bg-slate-900` for cards/panels
- **Borders:** 1px solid `border-slate-800` — no heavy shadows, rely on border definitions
- **Accent:** "SAP Blue" → `text-blue-500` / `bg-blue-600` for primary actions and active states
- **Typography:** `Inter` for all UI text; `font-mono` (JetBrains Mono) strictly for financial figures, inventory quantities, document numbers, and ID fields
- **Spacing:** `p-6` for card interiors; `gap-4` / `gap-6` for grid layouts — mathematically consistent throughout
- **Custom Tokens (tailwind.config.ts):** `--sap-blue`, `--alert-red`, `--stock-green`

## Verification Plan

### Automated Tests
- `npx tsc --noEmit` — verify all SAP payload types strict, no implicit `any`
- `npm run build` — confirm all App Router pages and route handlers compile

### Manual Verification

| Check | Expected Result |
|-------|----------------|
| Visit `/dashboard` without cookie | Redirected to `/login` |
| Login → inspect cookies in DevTools | `B1SESSION` is `HttpOnly`, `SameSite=Strict` |
| Click Logout | Cookie cleared, redirected to `/login` |
| `GET /api/sap/Orders?$filter=DocumentStatus eq 'bost_Open'` | Returns only open orders |
| `GET /api/sap/Items?$select=ItemCode,OnHand` | Returns only those two columns |
| `GET /api/sap/Orders?$orderby=DocDate desc` | Orders sorted newest first |
| `POST /api/sap/Orders` with valid body | Returns `201` with new order |
| Business Partners page → click "Vendors" chip | Only `CardType: 'V'` rows shown |
| Partner row where `Balance > CreditLimit` | Row tinted `bg-red-950/30`, red alert icon visible |
| Inventory item where `NetAvailable < MinStock` | Cell pulses red, "Restock Required" badge shown |
